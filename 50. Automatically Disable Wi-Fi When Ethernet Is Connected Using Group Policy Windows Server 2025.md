## Automatically Disable Wi-Fi When Ethernet Is Connected Using Group Policy | Windows Server 2025

## 📌 Overview

This document provides standardized procedures for configuring a Group Policy Object (GPO) in **Windows Server 2025** to automatically block wireless (Wi-Fi) connections when a domain-authenticated Ethernet connection is active.

This configuration ensures:

* 🔐 Improved network security
* 🛡 Prevention of simultaneous wired and wireless connections
* 🏢 Enforcement of corporate network access policies
* 📋 Compliance with IT governance standards

---

## 🎯 Scope

This procedure applies to:

* Domain-joined Windows client computers
* Organizational Units (OUs) containing target computers
* Managed environments under centralized Active Directory control

---

| Target    | Client Computers (OU linked)                                         |

# ⚙️ Implementation Procedure

---

## 🪜 Step 1: Open Group Policy Management

1. Open **Server Manager**.
2. Click **Tools** → **Group Policy Management**.

---

## 🪜 Step 2: Create a New Group Policy Object (GPO)

📍 Navigate to the Organizational Unit (OU) where the target users or computers are located.

### Example:

Domain: `xitiztechservices.local`

### 📂 Steps:

1. Go to **Group Policy Objects**.
2. Right-click → **New**.
3. Name the GPO:

```
Automatically Disable Wi-Fi When Ethernet Is Connected
```

4. Right-click **"Automatically Disable Wi-Fi When Ethernet Is Connected"** → Click **Edit**.

5. Navigate to:

```
Computer Configuration 
> Policies 
> Administrative Templates 
> Network 
> Windows Connection Manager
```

6. Select the policy setting:

```
Prohibit connection to non-domain networks when connected to domain authenticated network
```

7. Configure the setting:

   * Select **Enable**
   * Click **Apply**
   * Click **OK**

---

## 🔄 Step 3: Update Group Policy on Server

1. Open **PowerShell**.
2. Right-click → **Run as Administrator**.
3. Execute:

```powershell
gpupdate /force
```

---

## 💻 Step 4: Update Group Policy on Client Computer

1. Go to the client computer.
2. Open **PowerShell**.
3. Right-click → **Run as Administrator**.
4. Execute:

```powershell
gpupdate /force
```

5. Restart the computer.

1. Disable the Ethernet adapter.

**Expected Result:**
Wi-Fi will enabled automatically.

---

## ✅ Test 2 – Ethernet Enabled

1. Enable the Ethernet adapter.
2. Check the Wi-Fi status.

**Expected Result:**
Wi-Fi will disabled automatically.

# 🔎 Operational Notes

* Ensure the GPO is properly linked to the intended OU.
* Confirm client machines are domain-joined.
* Validate Active Directory replication if multiple Domain Controllers exist.
* Test in staging before production deployment.
* Use `gpresult /r` on client machines for troubleshooting.

---

# 🚨 Troubleshooting Tips

| Issue                 | Possible Cause                  | Resolution                       |
| --------------------- | ------------------------------- | -------------------------------- |
| Policy not applied    | GPO not linked                  | Verify OU linkage                |
| Wi-Fi not disabling   | Client not domain authenticated | Check domain connection          |
| Settings not updating | GP not refreshed                | Run `gpupdate /force` and reboot |
