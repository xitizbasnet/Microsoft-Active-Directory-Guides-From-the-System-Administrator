# 🖥️ Fix Remote Shutdown Not Supported Error Using Remote Shutdown Dialog from Domain – Windows Server 2025

## 📌 Overview

This document describes how to resolve the **“Remote Shutdown Not Supported”** error when attempting to remotely shut down a client computer from a **Domain Controller or management server** using the **Remote Shutdown Dialog** in **Windows Server 2025**.

The issue typically occurs when the **target client computer does not allow remote management services through Windows Defender Firewall**.

After applying the configuration described in this guide, administrators will be able to **successfully perform remote shutdown operations from the Active Directory environment**.

---

# 🧰 Environment

| Component       | Description                         |
| --------------- | ----------------------------------- |
| Server          | Windows Server 2025                 |
| Domain          | Active Directory Domain Environment |
| Client Example  | `TestSRV01`                         |
| Tool Used       | Remote Shutdown Dialog              |
| Command Utility | Command Prompt (Administrator Mode) |

---

# 🚀 Step 1 — Initiate Remote Shutdown from the Domain Server

1. Open **Command Prompt**.

2. Right-click **Command Prompt** and select:

```
Run as Administrator
```

3. Execute the following command:

```
shutdown -i
```

4. Press **Enter**.

5. The **Remote Shutdown Dialog** pop-up window will appear.

---

## Select Target Computer

1. Under **Computers**, click:

```
Browse
```

2. A new window will open:

```
Active Directory Computers
```

3. Click:

```
Find Now
```

4. Select the target computer.

Example:

```
TestSRV01
```

5. Click:

```
OK
```

---

## Configure Shutdown Options

In the **Remote Shutdown Dialog**, configure the following settings.

### Action

Under the option:

```
What do you want these computers to do?
```

Select:

```
Shutdown
```

---

### User Warning

Enable the option:

```
Warn users of the action
```

Set warning time:

```
30 seconds
```

---

### Shutdown Event Tracking

Enable:

```
Planned
```

From the dropdown menu choose:

```
Operating System: Recovery (Planned)
```

---

### Comment

Example comment:

```
Testing remote shutdown from AD to client
```

Click:

```
OK
```

---

# ⚠️ Error Message

After executing the command, the following error may appear:

```
TestSRV01: The entered computer name is not valid or remote shutdown is not supported on the target computer. 
Check the name and then try again or contact your system administrator. (53)
```

---

# 🔍 Cause of the Problem

This error occurs because the **target client computer blocks remote administrative services** through **Windows Defender Firewall**.

The following services must be allowed:

* Remote Service Management
* Windows Management Instrumentation (WMI)

---

# 🛠️ Step 2 — Fix the Issue on the Client Computer

Log in to the client computer.

Example:

```
TestSRV01
```

---

## Enable Required Firewall Rules

Navigate to:

```
Control Panel
```

Then open:

```
Windows Defender Firewall
```

Click:

```
Allow an app or feature through Windows Defender Firewall
```

---

### Enable Remote Service Management

Enable all entries for:

```
Remote Service Management
```

Make sure the following profiles are checked:

* Domain
* Private
* Public

---

### Enable Windows Management Instrumentation (WMI)

Enable all entries for:

```
Windows Management Instrumentation (WMI)
```

Make sure the following profiles are checked:

* Domain
* Private
* Public

---

### Apply Changes

Click:

```
OK
```

---

## 🔄 Reboot the Client Computer

Restart the system to ensure the firewall configuration is applied.

---

# 🔁 Step 3 — Retry Remote Shutdown from the Domain Server

Return to the **Active Directory Server**.

1. Open **Command Prompt**.

2. Right-click and select:

```
Run as Administrator
```

3. Execute:

```
shutdown -i
```

4. Press **Enter**.

The **Remote Shutdown Dialog** will appear again.

---

### Select Target Computer

Click:

```
Browse
```

Then:

```
Find Now
```

Select:

```
TestSRV01
```

Click:

```
OK
```

---

# ✅ Result

The client computer will now **successfully perform a remote shutdown** from the **Active Directory server** without the previous error.

---

# 📎 Administrator Notes

💡 Ensure the following requirements are met:

* Administrator must have **Domain Admin privileges**
* Client computer must be **reachable via network**
* Required firewall rules must be **enabled**
* Client computer must be **restarted after firewall changes**
